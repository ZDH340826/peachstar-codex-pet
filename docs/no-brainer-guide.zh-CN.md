# 无脑版：只靠对话生成自己的 Codex 宠物

这份教程给不想研究技术细节的人。你只需要准备素材、复制提示词、按 Codex 的回复继续确认，就可以生成一个自己的 Q 版宠物。

## 你需要准备什么

准备 3 到 8 张图片即可：

- 人物照片或宠物照片：用来提取长相、气质、发型、服饰。
- 风格参考图：例如 Q 版头像、像素风角色、可爱贴纸、游戏小人。
- 装饰参考图：例如帽子、衣服、主题色、配饰、IP 风格。

建议：

- 最少 1 张主体参考图 + 1 张风格参考图。
- 图片越清晰越好。
- 如果涉及真人照片，不建议把原图上传到公开仓库。

## 第一步：把素材发给 Codex

在 Codex 对话里上传图片，然后直接发这段：

```text
请根据我上传的素材，帮我生成一个 Codex 可用的 Q 版桌面宠物。

要求：
1. 主体是 Q 版人物/宠物/角色，不要做成写实图。
2. 风格参考我上传的风格图。
3. 保留主体最明显的识别点，例如发型、服装、颜色、表情、配饰。
4. 最终要生成 Codex pet 包，包含 pet.json 和 spritesheet.webp。
5. 请不要上传原始参考照片到 GitHub，只上传最终宠物资源和教程。
6. 如果需要使用子代理生成动作帧，我允许使用。
```

如果你想要类似本项目的 Q 版人物风，可以补一句：

```text
我想要 Q 版人物，不是动物身体。可以使用可爱帽兜、蝴蝶结、星星、浅色裙装等装饰。
```

## 第二步：告诉 Codex 你想要的风格

复制下面模板，把括号里的内容改掉：

```text
我的宠物设定如下：

名字：（例如 PeachStar / 小桃星 / Luna）
主体：（例如 Q 版女生 / 小猫 / 小机器人 / 小狐狸）
风格：（例如 软萌 Q 版 / 像素风 / 贴纸风 / 日系头像风）
主色：（例如 蓝白 / 粉白 / 奶黄 / 黑金）
关键元素：（例如 长棕发、毕业帽、星星、樱花、玉桂狗帽兜）
不要出现：（例如 写实脸、复杂背景、文字、水印、阴影、漂浮特效）

请先生成主形象，再生成完整 Codex pet 动作帧。
```

示例：

```text
我的宠物设定如下：

名字：PeachStar
主体：Q 版女生
风格：软萌 Q 版头像风
主色：蓝白、奶油黄、浅粉
关键元素：长棕发、大眼睛、玉桂狗帽兜、蓝色蝴蝶结、毕业帽星星、樱花腮印
不要出现：写实脸、动物身体、复杂背景、文字、水印、阴影、漂浮特效

请先生成主形象，再生成完整 Codex pet 动作帧。
```

## 第三步：确认主形象

Codex 会先生成一个主形象。你只需要判断：

- 像不像你想要的主体？
- 是不是 Q 版？
- 关键元素有没有保留？
- 有没有变成错误类型？例如你想要人物，它却做成动物。
- 有没有太复杂？小尺寸下是否还能看清？

如果不满意，直接说：

```text
方向不对，请重做主形象。我要的是（你的要求），不是（错误点）。
```

示例：

```text
方向不对。我要的是 Q 版人物，不是动物宠物。玉桂狗元素只作为帽兜装饰，主体仍然是女生。
```

满意后说：

```text
这个主形象可以，继续生成动作帧。
```

## 第四步：允许生成动作帧

Codex pet 通常需要多行动作帧。你可以直接回复：

```text
允许使用子代理继续生成动作帧。
```

常见动作包括：

- idle：待机
- running-right：向右移动
- running-left：向左移动
- waving：挥手
- jumping：跳跃
- failed：失败或沮丧
- waiting：等待
- running：处理中
- review：审阅或思考

## 第五步：让 Codex 修复问题

生成过程中可能出现：

- 某一行动作裁切了耳朵或帽子。
- 角色身份漂移。
- 背景不干净。
- 动作帧没有分开。
- 校验失败。

你不用自己修，直接说：

```text
请只修复失败的动作行，不要重做整个宠物。
```

如果你看到明显问题，也可以描述：

```text
第 running 行帽子被裁切了，请重生成这一行，保持角色居中、每帧完整、不要跨格。
```

## 第六步：安装宠物

完成后，Codex 会生成类似这样的目录：

```text
C:\Users\<你的用户名>\.codex\pets\<pet-name>\
```

里面应有：

```text
pet.json
spritesheet.webp
```

重启 Codex，然后在宠物或外观设置里选择这个宠物。

## 第七步：上传 GitHub

如果你想分享项目，可以让 Codex 上传到 GitHub：

```text
请将这个宠物项目提交到 GitHub。
要求：
1. 上传最终 pet.json 和 spritesheet.webp。
2. 上传 README、中文教程和 QA 预览图。
3. 不要上传原始参考照片。
4. 不要上传中间生成目录。
5. 仓库可以公开。
```

建议仓库结构：

```text
pets/
  your-pet-name/
    pet.json
    spritesheet.webp
docs/
  images/
  no-brainer-guide.zh-CN.md
README.md
README.zh-CN.md
.gitignore
```

`.gitignore` 建议包含：

```text
image/
pet-runs/
```

## 一句话版本

如果你什么都不想写，可以直接上传素材后发：

```text
请根据我上传的图片，帮我生成一个 Codex 可用的 Q 版宠物。你来决定风格、动作帧和文件结构。生成完成后，请打包为 pet.json + spritesheet.webp，并写一份中文 README 和无脑版教程。不要上传原始参考照片。
```
