# 如何制作自己的 Codex 宠物

这份文档整理了 PeachStar 的制作流程。它不是原始聊天逐字稿，而是可复用的操作手册：看完仓库后，应该能理解如何准备素材、设计角色、生成动作帧、校验并安装自己的 Codex 宠物。

## 1. 明确目标

先决定宠物的基础方向：

- 主体：动物、物品、Q 版人物、机器人、植物等。
- 风格：像素风、Q 版头像风、软萌风、机械风、简洁工具风等。
- 关键识别点：发型、帽子、衣服、颜色、配饰、表情、标志性动作。
- 是否公开：如果包含真人参考图，公开仓库时建议只上传最终成品，不上传原图。

PeachStar 的最终方向是：

- Q 版人物女孩，不是动物身体。
- 玉桂狗风格帽兜作为服饰装饰。
- 长棕发、蓝白配色、毕业帽星星、樱花腮印。
- 适配 Codex 小尺寸桌面宠物。

## 2. 准备参考图

建议准备两类图片：

- 人物或主题参考：用于提取气质、颜色、服饰、发型。
- 风格参考：用于约束 Q 版、像素、可爱程度、线条风格。

注意：

- 不建议把原始私密照片上传到公开仓库。
- 可以在仓库里只保留最终 `spritesheet.webp`、`pet.json` 和 QA 预览图。
- 如果参考的是有版权的角色，仅建议个人学习或个人使用，公开传播和商用要谨慎。

## 3. 设计主形象

先生成一个完整的主形象，它会成为后续所有动作帧的身份基准。

主形象提示词应包含：

```text
主体是什么：
  Q 版人物女孩，不是动物身体。

关键视觉：
  长棕发、大眼睛、蓝白帽兜、长耳朵、蓝色蝴蝶结、毕业帽星星、浅色裙子。

尺寸要求：
  全身、居中、适合 192 x 208 单帧。

风格要求：
  小尺寸可读、轮廓清楚、扁平上色、少细节。

背景要求：
  纯色绿幕或蓝幕，方便后续抠透明。

禁止项：
  不要文字、不要背景场景、不要阴影、不要漂浮特效、不要复杂小饰品。
```

PeachStar 的关键经验：

- “宠物”容易被理解成动物，所以如果你要 Q 版人物，必须明确写“chibi human girl, not an animal mascot”。
- 如果帽子、长耳、发饰很宽，后续动作帧容易裁切，主形象阶段就要留足边距。

## 4. 生成动作帧

Codex pet 使用固定图集：

- 总尺寸：`1536 x 1872`
- 网格：8 列 x 9 行
- 单帧：`192 x 208`
- 背景：透明
- 未使用格子：透明

PeachStar 包含 9 个动作状态：

| 行 | 状态 | 帧数 | 用途 |
| --- | --- | --- | --- |
| 0 | `idle` | 6 | 静止呼吸、眨眼 |
| 1 | `running-right` | 8 | 向右移动 |
| 2 | `running-left` | 8 | 向左移动 |
| 3 | `waving` | 4 | 打招呼 |
| 4 | `jumping` | 5 | 跳跃 |
| 5 | `failed` | 8 | 失败、沮丧 |
| 6 | `waiting` | 6 | 等待 |
| 7 | `running` | 6 | 工作中、处理中 |
| 8 | `review` | 6 | 审阅、思考 |

每一行动作帧都要满足：

- 一行横向排列。
- 每个槽位一个完整角色。
- 角色不要跨槽、不要裁切。
- 背景必须干净，便于转透明。
- 每帧都要保持同一个角色身份。
- 不要添加文字、UI、背景、速度线、漂浮星星、灰尘、阴影。

## 5. 处理非对称角色

如果角色左右不对称，不要简单镜像。

PeachStar 没有直接镜像 `running-right`，而是单独生成 `running-left`，原因是：

- 毕业帽和星星位置非对称。
- 蓝色蝴蝶结在一侧。
- 发饰和腮印在一侧。
- 直接镜像会让配件方向变错。

判断标准：

- 纯对称角色：可以考虑镜像。
- 有文字、标志、单侧配饰、单侧发型、单侧脸部标记：应单独生成。

## 6. 校验和修复

动作帧生成后，需要抽帧、抠透明、拼图集并校验。

合格标准：

- `spritesheet.webp` 是 `RGBA`。
- 尺寸是 `1536 x 1872`。
- 透明背景正确。
- 未使用格子透明。
- 无校验错误和警告。
- QA 接触表中所有动作行身份一致。

PeachStar 遇到的问题：

- `failed` 和 `running` 第一次生成时布局不稳定。
- 抽帧工具退化为按槽硬切。
- 解决方式：只重生失败的两行，不重做整个宠物。

修复提示词重点：

```text
Exactly N equal-width frames in one horizontal row.
Each frame contains one complete character.
Centered, similar scale, clear green gap between frames.
No overlap, no crossing, no clipped ears or hat.
```

## 7. 安装到 Codex

最终宠物目录需要长这样：

```text
peachstar/
  pet.json
  spritesheet.webp
```

`pet.json` 示例：

```json
{
  "id": "peachstar",
  "displayName": "PeachStar",
  "description": "A custom Codex digital pet.",
  "spritesheetPath": "spritesheet.webp"
}
```

复制到：

```text
C:\Users\<你的用户名>\.codex\pets\peachstar
```

然后重启 Codex，在宠物或外观设置里选择它。

## 8. 做放大版

Codex 的图集规格固定，不能直接把整张图改成更大尺寸。

如果想让屏幕上看起来更大，有两种方式：

- 优先使用 Codex 内部显示缩放设置，如果客户端支持。
- 复制一个新版本，在每个 `192 x 208` 单帧内部把角色内容放大并重新居中。

PeachStar Large 使用第二种方式。风险是角色长耳和帽子已经接近单帧边界，放大后部分帧安全余量更小。

## 9. 仓库发布建议

如果要把项目传到 GitHub：

建议上传：

- `pets/<pet-name>/pet.json`
- `pets/<pet-name>/spritesheet.webp`
- `README.md`
- `README.zh-CN.md`
- `docs/images/contact-sheet.png`
- 脱敏制作过程文档

建议忽略：

```text
image/
pet-runs/
```

原因：

- `image/` 可能包含真人照片或私密参考图。
- `pet-runs/` 可能包含中间生成文件、本机路径和大量临时文件。

本仓库采用的策略就是只上传最终可安装宠物包和脱敏教程。
